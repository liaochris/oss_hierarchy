      📚 The doc issue

      PyTorch Leaky ReLU documentation

      As the author in this post critizises, the naming "negative slope" for the parameter in Leaky ReLU is somewhat confusing as it is not a "negative" slope (in the narrow sense, see here) but refers to the negative part (quadrant III in a cartesian coordinate system of the slope of the input activated by the Leaky ReLU function.

      For comparison, in TensorFlow/Keras the parameter is called alpha:

      alpha | Float >= 0. Negative slope coefficient. Default to 0.3.

      TensorFlow Leaky ReLU documentation

      Suggest a potential alternative/fix

      • Adaption from TF: alpha
      • Update documentation with more precise information, that alpha refers to the negative part (quadrant III in a cartesian coordinate system) of the slope
