\section{Model}

\begin{frame}{Baseline event study}
\small
\begin{equation*}
Y_{i,t}^{SD} = \alpha_i + \lambda_t + \sum_{e \neq \infty}\sum_{k \neq -1} \delta_{e,k} Z_{i,t}^{e,k} + \varepsilon_{i,t}
\end{equation*}

\begin{itemize}
  \item $Y_{i,t}^{SD}$ is a standardized outcome: pull requests opened, pull requests merged, or software releases. Standardization uses the five most recent pre-periods, so coefficients are in standard-deviation units.
  \item $Z_{i,t}^{e,k}$ equals one when organization $i$ belongs to departure cohort $e$ and is at event time $k$ in period $t$.
  \item $\alpha_i$ and $\lambda_t$ are organization and calendar-time fixed effects.
  \item The weighted average of the cohort-specific coefficients, $\hat{\delta}_k$, is average resilience $k$ periods after departure under parallel trends and no anticipation.
\end{itemize}
\note{
The original outline wanted the benchmark model and the parameter that maps to resilience.
Make the identifying assumptions explicit when introducing the interpretation of $\hat{\delta}_k$.
}
\end{frame}

\begin{frame}{Why abrupt departures are plausibly exogenous}
\small
\begin{itemize}
  \item Existing survey evidence from Miller, ``Why Do Modern Open Source Contributors Become Inactive?'' points to job changes, graduation, marriage, and children as common reasons for abrupt exit.
  \item Those are individual life transitions, not obvious responses to latent organizational decline or dissatisfaction with project leadership.
  \item The no-anticipation assumption is also more plausible with six-month periods: organizations would need substantial advance notice to change behavior far enough in advance to matter in the pre-periods.
  \item This does not make exogeneity automatic, but it makes abrupt timing more credible than broader notions of gradual disengagement.
\end{itemize}
\note{
This is the exogeneity slide from the original outline.
The Miller survey evidence is the anchor.
If you later add more direct validation for abrupt-departure exogeneity, it slots here or in backup.
}
\end{frame}

\begin{frame}{Why move to machine learning?}
\small
\begin{equation*}
Y_{i,t}^{SD} = \alpha_i + f_t(X_i) + \sum_{e \neq \infty}\sum_{k \neq -1} \delta_{e,k}(X_i) Z_{i,t}^{e,k} + \varepsilon_{i,t}
\end{equation*}

\begin{itemize}
  \item Median splits hard-code one-practice-at-a-time thresholds and miss nonlinearities and interactions across practices.
  \item The generalized random forest event study lets baseline outcomes vary flexibly with practices through $f_t(X_i)$ and lets resilience itself vary with practices through $\delta_{e,k}(X_i)$.
  \item For prediction, the object of interest is the hold-out post-period effect $\hat{\psi}(X_i)$.
  \item For interpretation, the object of interest is how combinations of practices move $\hat{\psi}(X_i)$ and therefore change the high-versus-low resilience split.
  \item The key identifying assumptions are conditional parallel trends and conditional no anticipation.
\end{itemize}
\note{
This slide should distinguish prediction from interpretation, exactly as in the original outline.
Prediction is about the hold-out effect estimates.
Interpretation is about how the practice vector moves those estimates.
}
\end{frame}

\begin{frame}{Combining practices out of sample}
\small
\begin{enumerate}
  \item Estimate the generalized random forest event study using pre-departure organizational practice scores and pull requests merged as the training outcome.
  \item Cross-fit the forest so that each organization's resilience prediction is out of sample.
  \item Compute the doubly robust average post-period treatment effect, $\hat{\psi}(X_i)$, for each organization.
  \item Classify organizations above the median of $\hat{\psi}(X_i)$ as high resilience and those below the median as low resilience.
\end{enumerate}

\begin{frame}{Benchmark and robustness checks}
\small
\begin{itemize}
  \item \textbf{Benchmark:} compare the forest split with the oracle or perfect-prediction grouping and report the share of the oracle gap recovered.
  \item \textbf{Fold robustness:} rank all 32 coarsened practice combinations within each fold, keep the ordering fixed, and report alignment or average pairwise correlations across folds.
  \item \textbf{Departure-timing robustness:} test sensitivity to the abrupt-departure assumption and verify that nearby organizational outcomes are not moving before departure.
  \item \textbf{Pre-trend robustness:} add Rambachan-Roth style sensitivity bounds for the main event-study comparisons.
  \item \textbf{Dynamics:} compare short-run and long-run resilience groups to see whether the same practices support both.
\end{itemize}
