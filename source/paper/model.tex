\section{Model}
\subsection{Organizational workflow}

Each time period $t$, project $i$ faces a latent number of problems $N_{i,t} \sim F_{it}(\cdot)$, where $F_{it}$ is a discrete probability mass function.
Each problem passes through up to three stages within the period: 
\begin{enumerate}
    \item a latent problem ($\ell$) may lead to an opened pull request ($o$) 
    \item an opened pull request ($o$) may be reviewed ($r$)
    \item a reviewed pull request ($r$) may be merged ($m$); state $m$ is absorbing.
\end{enumerate}
Let $X_{n,t} \in \{\ell,o,r,m\}$ denote the state of problem $n$ within period $t$.

I characterize state transitions under three assumptions.

\begin{assumption}[Problem exchangeability]
\label{asm:exchangeability}
Problems are indistinguishable: for any two problems $n$ and $n'$ and states $x,y \in \{\ell,o,r,m\}$, the probability that a problem in state $x$ transitions to state $y$ is the same for $n$ and $n'$. Conditional on being in the same state, any two problems have the same probability of advancing to the next state.
\end{assumption}

\begin{assumption}[Problem independence]
\label{asm:independence}
Conditional on their current states, problems transition independently of one another.
\end{assumption}

\begin{assumption}[Memoryless state transitions]
\label{asm:memoryless}
A problem's transition probability from state $x$ depends only on $x$, not on how it arrived at $x$.
\end{assumption}

Under Assumptions~\ref{asm:exchangeability}--\ref{asm:memoryless}, every problem in project $i$ at time $t$ faces the same transition matrix
$$
\pi_{i,t} =
\begin{pmatrix}
1-\pi_{i,t}^o & \pi_{i,t}^o & 0 & 0 \\
0 & 1-\pi_{i,t}^r & \pi_{i,t}^r & 0 \\
0 & 0 & 1-\pi_{i,t}^m & \pi_{i,t}^m \\
0 & 0 & 0 & 1
\end{pmatrix},
$$
where $\pi_{i,t}^o$ is the probability that a latent problem leads to an opened pull request, $\pi_{i,t}^r$ is the probability that an opened pull request is reviewed, and $\pi_{i,t}^m$ is the probability that a reviewed pull request is merged.

The expected counts at each stage 
\begin{enumerate}
    \item for pull requests opened $N_{i,t}^o$ is $\mathbb{E}[N_{i,t}^o] = \mathbb{E}[N_{i,t}]\cdot\pi_{i,t}^o$,
    \item for pull requests reviewed $N_{i,t}^r$ is $\mathbb{E}[N_{i,t}^r] = \mathbb{E}[N_{i,t}^o]\cdot\pi_{i,t}^r$, and 
    \item for pull requests merged $N_{i,t}^m$ is $\mathbb{E}[N_{i,t}^m] = \mathbb{E}[N_{i,t}^r]\cdot\pi_{i,t}^m$. 
\end{enumerate}

\subsection{Individual contributions to organizational outcomes}

My goal is to predict how organizational activity changes when a member leaves. To do that, I need a model of how individual actions contribute to organizational outcomes.

Let $\mathcal{J} = \{1, \ldots, J\}$ be the set of members. Let $A_{j,t}^o$, $A_{j,t}^r$, and $A_{j,t}^m$ denote the counts of pull requests opened, reviewed, and merged by member $j$ in period $t$. For state $s \in \{o, r, m\}$, let $p_{j,t}^s \in [0,1]$ denote member $j$'s probability of performing a state-$s$ action on any given unit at time $t$. For a given latent problem, at most one member can open and merge a pull request; any number of members can review an opened pull request. 

\begin{assumption}[Problem-invariant member probabilities]
\label{asm:problem-invariant}
A member's probability of opening, reviewing, or merging a pull request does not vary across problems within a period --- $p_{j,t}^s$ is the same regardless of which particular problem is being acted on.
\end{assumption}

For any latent problem in time period $t$, let $p_{j,t}^o$ denote the probability that member $j$ is the unique member who opens a pull request to address that problem. The probability that no member opens a pull request is $p_{0}^o = 1-\sum_{j\in\mathcal J} p_{j,t}^o$.
Under Assumptions~\ref{asm:independence} and~\ref{asm:problem-invariant}, the identity of the opening member is an i.i.d.\ draw from a categorical distribution over $\mathcal{J} \cup \{0\}$ across problems, so the joint count vector follows a multinomial:
$$ \left( \left\{A_{j,t}^o \right\}_{j\in\mathcal J}, A_{0, t}^o \right) \mid N_{i,t} \sim \text{Multinomial}\left(N_{i,t}; \left \{p_{j,t}^o \right \}_{j\in\mathcal J}, p_{0}^o \right). $$
Hence, $\pi_{i,t}^o = \sum_{j\in\mathcal J}p_{j,t}^o$, and the number of pulls opened is distributed binomially as $N_{i,t}^o \mid N_{i,t} \sim \text{Binomial}(N_{i,t},\, \pi_{i,t}^o)$.


\begin{assumption}[Independent review decisions]
\label{asm:review-independence}
Conditional on an opened pull request, each member $j$ decides whether to review it independently of all other members, so member $j$'s reviews decision does not affect member $j'$ decision. 
\end{assumption}

Under Assumptions~\ref{asm:problem-invariant} and~\ref{asm:review-independence}, conditional on $N_{i,t}^o$ opened pull requests, member $j$ independently reviews each with probability $p_{j,t}^r$. The quantity of pull requests reviewed by member $j$ in period $t$ is the sum of $N_{i,t}^o$ independent Bernoulli actions,
$$A_{j,t}^r = \sum_{n=1}^{N_{i,t}^o}\mathbf{1}[\text{member }j\text{ reviews pull request }n],$$
so $A_{j,t}^r \mid N_{i,t}^o \sim \text{Binomial}(N_{i,t}^o, p_{j,t}^r)$. 
Hence, the probability that at least one member reviews any given pull request is $\pi_{i,t}^r = 1-\prod_{j\in\mathcal J}(1-p_{j,t}^r)$.

For any reviewed pull request at time $t$, let $p_{j,t}^m$ denote the probability that member $j$ is the unique member who merges it. 
The probability that no member merges it is $p_{0}^m = 1-\sum_{j\in\mathcal J} p_{j,t}^m$.
Under Assumptions~\ref{asm:independence} and~\ref{asm:problem-invariant}, by the same independence argument as for opening:
$$ \left( \left \{A_{j,t}^m\right \}_{j\in\mathcal J}, A_{0, t}^m \right) \mid N_{i,t}^r \sim \text{Multinomial}\left(N_{i,t}^r; \left \{p_{j,t}^m\right \}_{j\in\mathcal J}, p_{0,t}^m \right). $$
Hence, $\pi_{i,t}^m = \sum_{j\in\mathcal J}p_{j,t}^m$, and the number of pulls merged is distributed binomially as $N_{i,t}^m \mid N_{i,t}^r \sim \text{Binomial}(N_{i,t}^r,\, \pi_{i,t}^m)$.


\subsection{Fitting model parameters}
I next fit the model parameters using pre-period data prior to $j^*$'s departure, so I can use the model to simulate draws from the implied DGP. 
I do so using the observed data, which is at best, a single draw from the model-implied DGP. 
I make four assumptions about the model to simplify estimation. 

\begin{assumption}[Complete problem opening]
\label{asm:complete-opening}
In pre-periods $t$, $p_{0,t}^o = 0$: every latent problem leads to an opened pull request. 
Under this assumption, $N_{i,t} = N_{i,t}^o$, so the quantity of latent problems is observed in the data as the quantity of pull requests opened. 
\end{assumption}

\begin{assumption}[Time-invariant latent flow]
\label{asm:time-invariant}
$F_{i,t} = F_i$ for all $t$: pre-period opening counts $N_{i,t}^o$ are i.i.d.\ draws from a time-invariant organization-specific distribution $F_i$.
\end{assumption}

\begin{assumption}[Stationary member probabilities]
\label{asm:stationary}
$p_{j,t}^s = p_j^s$ for all $t$ and $s$: member action probabilities do not vary across pre-periods. 
\end{assumption}

\begin{assumption}[Fixed membership]
\label{asm:fixed-membership}
The membership set $\mathcal{J}_{pre}$ is fixed across pre-periods, and the only change between the pre-period and post-period membership set is the focal departure $j^*$. Hence, $\mathcal{J}_{post} \setminus \mathcal{J}_{pre} = j^*$. 
\end{assumption}

I fit $F_i$ as either Poisson or Negative Binomial across all organizations, using organization-specific parameters estimated by maximum likelihood.

I estimate $p_j^s$ under either a pooled or per-period approach, applied uniformly across all organizations:
\begin{enumerate}
    \item \textbf{Pooled}: pool all pre-period observations; MLE gives the analytical solution $\hat{p}_j^o = \frac{\sum_t A_{j,t}^o}{\sum_t N_{i,t}^o}$, $\hat{p}_j^m = \frac{\sum_t A_{j,t}^m}{\sum_t N_{i,t}^r}$, and $\hat{p}_j^r = \frac{\sum_t A_{j,t}^r}{\sum_t N_{i,t}^o}$.
    \item \textbf{Per-period}: Treat the period-specific estimates
    $\{\hat{p}_{j,t}^s\}_{t=1}^T$ as repeated observations of a single underlying member-level probability parameter $\hat{p}_j^s$. The MLE for this common parameter is therefore the sample mean across pre-periods: $\hat{p}_j^s = \frac{1}{T}\sum_{t=1}^T \hat{p}_{j,t}^s$.
\end{enumerate}
Model selection across the Poisson/NB $\times$ pooled/per-period combinations will be addressed separately.

\subsection{Evaluating model fit on training data}

Although the model describes a DGP, the data represents only a few draws from that DGP. 
To determine how well the model describes the data, I compare the data to simulated data drawn from the model.

\subsubsection{Drawing from the model}

For each org $i$ and period $t$, I draw $K=1000$ outcome realizations from the fitted model.
Using the fitted latent problem flow distribution and stage-completion probabilities,

\begin{algorithm}[h]
\caption{DrawCounts($\hat{F}_i,\;\hat{\pi}^o,\;\hat{\pi}^r,\;\hat{\pi}^m$)}
\label{alg:drawcounts}
\begin{algorithmic}[1]
\For{$k = 1$ \textbf{to} $K$}
  \State Draw $N^{(k)} \sim \hat{F}_i$
  \State Draw $N^{o,(k)} \sim \mathrm{Binomial}(N^{(k)},\;\hat{\pi}^o)$ 
  \State Draw $N^{r,(k)} \sim \mathrm{Binomial}(N^{o,(k)},\;\hat{\pi}^r)$ 
  \State Draw $N^{m,(k)} \sim \mathrm{Binomial}(N^{r,(k)},\;\hat{\pi}^m)$ 
\EndFor
\State \Return $\bigl\{N^{o,(k)},\;N^{r,(k)},\;N^{m,(k)}\bigr\}_{k=1}^K$
\end{algorithmic}
\end{algorithm}

\subsubsection{Metric}
\label{sec:metric}

For a stage-count outcome $x \in \{N^o, N^r, N^m\}$, let $\{x_i^{(k)}\}_{k=1}^K$ denote the $K$ draws of $x$ from Algorithm~\ref{alg:drawcounts} for org $i$ --- since the model is time-invariant (Assumptions~\ref{asm:time-invariant}--\ref{asm:stationary}), these draws do not depend on $t$. Define the draw-implied mean and standard deviation
$$\mu_i(x) = \frac{1}{K}\sum_{k=1}^K x_i^{(k)}, \qquad \sigma_i(x) = \left(\frac{1}{K}\sum_{k=1}^K \bigl(x_i^{(k)} - \mu_i(x)\bigr)^2\right)^{1/2}.$$
For any observed value $x_{i,t}^{\mathrm{obs}}$ in period $t$, define
$$Q_{i,t}(x) = \left(\frac{x_{i,t}^{\mathrm{obs}} - \mu_i(x)}{\sigma_i(x)}\right)^2 - 1.$$
When the model correctly specifies the mean and variance of $x$, $\mathbb{E}[Q_{i,t}(x)] = 0$.
More precisely, if $G$ is the true distribution governing the data, $\mathbb{E}_G[Q_{i,t}(x)] = [(\mu_G - \mu_i(x))^2 + \sigma_G^2 - \sigma_i(x)^2]/\sigma_i(x)^2$, which is zero if and only if $\mu_G = \mu_i(x)$ and $\sigma_G = \sigma_i(x)$.
Each $Q_{i,t}(x)$ is a \emph{prediction error} --- the period-$t$ realization that allows us to diagnose \emph{misspecification error}, the systematic discrepancy between the model's implied distribution and the true DGP; aggregating prediction errors across periods provides a measure of misspecification error.


\subsubsection{Evaluating model fit}

The \emph{in-sample (IS)} approach fits the model on all $T$ pre-periods, and evaluates model fit $Q$ by comparing draws from Algorithm~\ref{alg:drawcounts} to each of the $T$ pre-period observations. The procedure is described in full in Algorithm~\ref{alg:evalinsample}.

\begin{algorithm}[h]
\caption{EvalInSample(org $i$)}
\label{alg:evalinsample}
\begin{algorithmic}[1]
\State Fit $\hat{F}_i$ and $\{\hat{p}_j^s\}$ on all $T$ pre-periods; compute $\hat{\pi}_i^s$.
\State Call Algorithm~\ref{alg:drawcounts} once using $(\hat{F}_i,\;\hat{\pi}_i^o,\;\hat{\pi}_i^r,\;\hat{\pi}_i^m)$, and compute $\mu_i(x)$ and $\sigma_i(x)$ for each $x \in \{N^o, N^r, N^m\}$.
\For{each pre-period $t \in \{1,\ldots,T\}$}
  \State $Q_{i,t}(x) = \left(\dfrac{x_{i,t}^{\mathrm{obs}} - \mu_i(x)}{\sigma_i(x)}\right)^2 - 1.$
\EndFor
\State Aggregate: $\bar{Q}_i^{IS}(x) = \dfrac{1}{T}\displaystyle\sum_{t=1}^T Q_{i,t}(x)$.
\end{algorithmic}
\end{algorithm}

$\bar{Q}_i^{IS}(x) \approx 0$ indicates the fitted model's distribution covers the pre-period data; $\bar{Q}_i^{IS}(x) \gg 0$ indicates systematic misspecification error even on training data.

\subsubsection{Outcomes and stage decomposition}

Algorithm~\ref{alg:stageoutcomes} computes three variants of $Q_{i,t}(N^m)$ that together allow me to decompose prediction error in the pull requests merged outcome into prediction error caused by misspecifying the opening and review stages. Each variant conditions progressively on observed intermediate counts, isolating misspecification error attributable to each stage.

\begin{algorithm}[h]
\caption{StageOutcomes(org $i$, period $t$)}
\label{alg:stageoutcomes}
\begin{algorithmic}[1]
\Statex \textbf{Input}: draws $\{N^{m,(k)}\}_{k=1}^K$ from Algorithm~\ref{alg:evalinsample}; observed $N_{i,t}^{o,\mathrm{obs}}$, $N_{i,t}^{r,\mathrm{obs}}$, $N_{i,t}^{m,\mathrm{obs}}$
\State Compute $Q_{i,t}(N^m)$ using $\{N^{m,(k)}\}$ and $N_{i,t}^{m,\mathrm{obs}}$.
\State Draw $\tilde{N}^{r,(k)} \sim \mathrm{Binomial}(N_{i,t}^{o,\mathrm{obs}},\;\hat{\pi}_i^r)$, $\tilde{N}^{m,(k)} \sim \mathrm{Binomial}(\tilde{N}^{r,(k)},\;\hat{\pi}_i^m)$.
\Statex \hspace{1.5em}$\hookrightarrow$\;Compute $Q_{i,t}(N^m \mid \mathrm{fix}\;N^o)$ using $\{\tilde{N}^{m,(k)}\}$ and $N_{i,t}^{m,\mathrm{obs}}$.
\State Draw $\hat{N}^{m,(k)} \sim \mathrm{Binomial}(N_{i,t}^{r,\mathrm{obs}},\;\hat{\pi}_i^m)$.
\Statex \hspace{1.5em}$\hookrightarrow$\;Compute $Q_{i,t}(N^m \mid \mathrm{fix}\;N^o,N^r)$ using $\{\hat{N}^{m,(k)}\}$ and $N_{i,t}^{m,\mathrm{obs}}$.
\end{algorithmic}
\end{algorithm}

Step~1 computes the full-model merge prediction error using draws from Algorithm~\ref{alg:evalinsample}. Step~2 conditions on actual opens $N_{i,t}^{o,\mathrm{obs}}$, so the prediction error reflects only review- and merge-stage misspecification. Step~3 additionally conditions on actual reviews $N_{i,t}^{r,\mathrm{obs}}$, so the prediction error reflects only merge-stage misspecification.

These yield an exact additive decomposition:
\begin{align*}
\Delta Q_{\mathrm{open}} &= Q_{i,t}(N^m) - Q_{i,t}(N^m \mid \mathrm{fix}\;N^o) \\
\Delta Q_{\mathrm{review}} &= Q_{i,t}(N^m \mid \mathrm{fix}\;N^o) - Q_{i,t}(N^m \mid \mathrm{fix}\;N^o,N^r) \\
Q_{\mathrm{merge}} &= Q_{i,t}(N^m \mid \mathrm{fix}\;N^o,N^r)
\end{align*}
so $\Delta Q_{\mathrm{open}} + \Delta Q_{\mathrm{review}} + Q_{\mathrm{merge}} = Q_{i,t}(N^m)$ exactly. $\Delta Q_{\mathrm{open}}$ measures how much of the total $N^m$ prediction error is attributable to getting $N^o$ wrong; $\Delta Q_{\mathrm{review}}$ measures prediction error from the review stage given correct opens; $Q_{\mathrm{merge}}$ is the residual prediction error from the merge stage given correct opens and reviews.

\subsection{Evaluating predictions on control and treated organizations}

I next evaluate how well the model fitted on all $T$ pre-periods predicts out-of-sample outcomes. 
Evaluating predictions on post-period data from control organizations allows me to distinguish departure-specific prediction error from prediction error due to model misspecification.
This is because control organizations do not experience departures, so their departure-specific prediction error is mechanically 0.
Evaluations of outcome prediction for treated organizations do mix departure-specific and misspecification error.
I also propose an approach for isolating departure-specific prediction error for treated organizations from misspecification error.

\subsubsection{Control organizations}

\begin{enumerate}
    \item For each control organization $i$, call Algorithm~\ref{alg:drawcounts} using $(\hat{F}_i,\;\hat{\pi}_i^o,\;\hat{\pi}_i^r,\;\hat{\pi}_i^m)$ which is fitted using pre-period data. Then, compute $\mu_i(N^m)$ and $\sigma_i(N^m)$.
    \item For each post-period $t'$, compute
    $$Q_{i,t'}^{post}(N^m) = \left(\frac{N_{i,t'}^{m,\mathrm{obs}} - \mu_i(N^m)}{\sigma_i(N^m)}\right)^2 - 1,$$ and calculate the average $\bar{Q}_i^{post}(N^m) = \frac{1}{T'}\sum_{t'=1}^{T'} Q_{i,t'}^{post}(N^m)$.
\end{enumerate}
$\bar{Q}_i^{post}(N^m)$ measures whether the model's predictive distribution for $N^m$ --- estimated entirely on pre-period data --- continues to describe post-period outcomes. The model assumes time-invariant $F_i$ and $p_j^s$ (Assumptions~\ref{asm:time-invariant}--\ref{asm:stationary}), hence abstracting away from systematic temporal variation.
$\bar{Q}_i^{post}(N^m) \approx 0$ validates this stationarity out of sample, while $\bar{Q}_i^{post}(N^m) \gg 0$ indicates the model has large misspecification error post-period.
Algorithm~\ref{alg:stageoutcomes} can additionally be applied to $Q_{i,t'}^{post}(N^m)$ to decompose post-period misspecification error by stage.

\subsubsection{Treated organizations}

\begin{enumerate}
    \item Compute counterfactual stage-completion probabilities after removing $j^*$:
    $$\hat{\pi}_i^{o,(-j^*)} = \sum_{j\in\mathcal J\setminus\{j^*\}}\hat{p}_j^o, \qquad \hat{\pi}_i^{r,(-j^*)} = 1-\prod_{j\in\mathcal J\setminus\{j^*\}}(1-\hat{p}_j^r), \qquad \hat{\pi}_i^{m,(-j^*)} = \sum_{j\in\mathcal J\setminus\{j^*\}}\hat{p}_j^m,$$
    where $\hat{\pi}_i^{o,(-j^*)}$ is the probability any latent problem gets opened absent $j^*$, $\hat{\pi}_i^{r,(-j^*)}$ is the probability any opened pull request receives at least one review, and $\hat{\pi}_i^{m,(-j^*)}$ is the probability any reviewed pull request gets merged.
    Note that the remaining members' probabilities are unchanged after the departure of $j^*$, which follows directly from Assumption~\ref{asm:stationary}.
    \item Call Algorithm~\ref{alg:drawcounts} using $(\hat{F}_i,\;\hat{\pi}_i^{o,(-j^*)},\;\hat{\pi}_i^{r,(-j^*)},\;\hat{\pi}_i^{m,(-j^*)})$, which is fitted on pre-period data. Then, compute $\mu_i^{(-j^*)}(N^m)$ and $\sigma_i^{(-j^*)}(N^m)$.
    \item For the post-departure period $t'$, compute
    $$Q_{i,t'}^{post}(N^m) = \left(\frac{N_{i,t'}^{m,\mathrm{obs}} - \mu_i^{(-j^*)}(N^m)}{\sigma_i^{(-j^*)}(N^m)}\right)^2 - 1$$, and calculate the average $\bar{Q}_i^{post}(N^m) = \frac{1}{T'}\sum_{t'=1}^{T'} Q_{i,t'}^{post}(N^m)$.
    \item Compute the differenced residual $\tilde{Q}_{i,t'}^{treated} = Q_{i,t'}^{post}(N^m) - \bar{Q}_i^{pre}$, where $\bar{Q}_i^{pre} = \frac{1}{T}\sum_{t=1}^T Q_{i,t}(N^m)$ is the in-sample pre-period baseline from Algorithm~\ref{alg:evalinsample}.
\end{enumerate}
$Q_{i,t'}^{post}(N^m) \approx 0$ indicates the counterfactual model accurately predicts post-departure $N^m$; $Q_{i,t'}^{post}(N^m) \gg 0$ indicates the model mis-predicts the impact of $j^*$'s departure, although we cannot say how much of this is due to mismodelling the effect of departure itself versus the model poorly fitting the data.

$\tilde{Q}_{i,t'}^{treated}$ attempts to solve this by removing the organization-specific baseline misspecification error that is present regardless of the departure.
Here, we assume that the model's pre-period misspecification error $\bar{Q}_i^{pre}$ is a good estimate of the model's general out-of-sample misspecification error for organization $i$.
$\tilde{Q}_{i,t'}^{treated} \approx 0$ indicates post-period misspecification error is no worse than pre-period misspecification error, whereas $\tilde{Q}_{i,t'}^{treated} \gg 0$ indicates the departure generates prediction error beyond what the model already had in the pre-period, suggesting that the model misspecifies the departure effect.
Algorithm~\ref{alg:stageoutcomes} can additionally be applied to $Q_{i,t'}^{post}(N^m)$ and $\tilde{Q}_{i,t'}^{treated}$ to decompose post-departure misspecification error by stage.

\Delta_m^{(j)}
\equiv
\pi_{i,t}^m - \pi_{i,t}^{m,(-j)}
=
p_{j,t}^{m}\prod_{j\in\mathcal J\setminus\{j\}}(1-p_{j,t}^{m}).
$$

Suppose member $j$ leaves before period $t+1$ and is not replaced, and suppose $N_{i,t}=N_{t+1}=N$. Then
$$
E[Q_{t+1}^m \mid N]
=
N \pi_{i,t}^{o,(-j)} \pi_{i,t}^{r,(-j)} \pi_{i,t}^{m,(-j)}
=
N(\pi_{i,t}^o-\Delta_o^{(j)})(\pi_{i,t}^r-\Delta_r^{(j)})(\pi_{i,t}^m-\Delta_m^{(j)}).
$$
Therefore,
$$
E[Q_{t+1}^m \mid N] - E[Q_t^m \mid N]
=
-N\Big[
\Delta_o^{(j)}(\pi_{i,t}^r-\Delta_r^{(j)})(\pi_{i,t}^m-\Delta_m^{(j)})
+\Delta_r^{(j)}\pi_{i,t}^o(\pi_{i,t}^m-\Delta_m^{(j)})
+\Delta_m^{(j)}\pi_{i,t}^o \pi_{i,t}^r
\Big].
$$

The three terms correspond to member $j$'s marginal contribution at the opening, review, and merge stages:
\begin{enumerate}
    \item $-N\,\Delta_o^{(j)}(\pi_{i,t}^r-\Delta_r^{(j)})(\pi_{i,t}^m-\Delta_m^{(j)})$ captures the loss from problems that would have been opened because member $j$ acts at the opening stage, and that would then have been reviewed and merged without member $j$'s involvement at those later stages.

    \item $-N\,\pi_{i,t}^o\Delta_r^{(j)}(\pi_{i,t}^m-\Delta_m^{(j)})$ captures the loss from pull requests that would have been opened (by anyone), then reviewed because member $j$ acts at the review stage, and then merged without member $j$'s involvement at the merge stage.

    \item $-N\,\pi_{i,t}^o \pi_{i,t}^r \Delta_m^{(j)}$ captures the loss from pull requests that would have been opened and reviewed (by anyone), and then merged because member $j$ acts at the merge stage.
\end{enumerate}

\subsection{Extensions}

\begin{enumerate}
    \item \textbf{Relax Assumption 1.} Problems need not come from an identical distribution. For example, the identity of the pull request opener may affect the probability that the pull request is eventually merged.

    \item \textbf{Relax Assumption 2.} Problems need not evolve independently. For example, one pull request being merged may increase the likelihood that followup problems are opened or closed.

    \item \textbf{Relax Assumption 4.} Member-specific probabilities may vary over time. For example, one could model $\pi_{i,t}^{o}$, $\pi_{i,t}^{r}$, and $\pi_{i,t}^{m}$ as draws from member-specific distributions, although this may require more data than are currently available.

    \item \textbf{Relax Assumption 5.} Members' actions within a stage need not be independent. For example, reviewers may coordinate so that one person's review makes another person's review less likely, or a member's departure may induce other members to increase their probabilities of acting. This extension would replace the independent-Bernoulli benchmark with a model that allows for coordination and/or organizational adaptation.
\end{enumerate}
